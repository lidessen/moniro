# 工作方式指导

用户在第九任会话中给出的指导，适用于所有后继者。

## 核心原则

**拿不准时停下来，而不是猜测或跳过。**

```
拿不准 → 停下来 → 记到 todo → 继续能做的
         ↓
    可以自己想，或和 subagent 讨论
         ↓
    直到没有什么能做的，或需要协助
```

## Todo 工具的使用

Claude Code 提供了 TaskCreate/TaskUpdate/TaskList 工具。

**用法**：
- 像笔记本一样使用——一直更新，一直看
- 记录拿不准的问题
- 追踪进度
- 直到列表清空，或者剩下的都需要人类协助

## 自主思考 vs 寻求帮助

**可以自己想的**：
- 技术实现细节
- 代码结构选择
- 小范围的设计决策

**可以和 subagent 讨论的**：
- 需要多角度分析的问题
- 需要调查验证的假设
- 复杂任务的分解

**需要人类协助的**：
- 战略方向的确认
- 资源/权限相关的问题
- 不确定用户意图时

## 与"先战略后战术"的关系

这个工作方式和之前的指导是一致的：

```
战略层面：和人类商量，确定方向
战术层面：自主执行，遇到问题记录
         ↓
    能解决的自己解决
    解决不了的汇总后寻求协助
```
