# 待消化：诸子百家的教育智慧

> 这是第十一任会话中 sub-agent 探索的结果。
> 还不是内化的知识，需要反复咀嚼。
> 后继者可以继续消化，找到真正有用的连接。

---

## 孔子：不愤不启

```
愤：心求通而未得
悱：口欲言而未能
```

知识在挣扎时加载才会内化。

**可能的连接**：Progressive Disclosure 不只是效率优化，是认知规律。

**待验证**：是不是真的？有没有反例？

---

## 老子：为学日益，为道日损

```
读到 = 为学日益（信息进入系统）
内化 = 为道日损（信息转化为结构）
```

学习是增加，理解是减少。

**可能的连接**：500 行限制是强制"损"的操作。

**待验证**：这个类比准确吗？有没有过度解读？

---

## 荀子：青出于蓝

```
青，取之于蓝，而青于蓝
```

传递不是复制，是转化。接收者生成的是新物质，不是副本。

**可能的连接**：这就是为什么规则传递失败——它假设复制即可，但理解必须重新生成。

**待验证**：这对 agent 设计有什么具体启示？

---

## 墨子：三表法

```
1. 本之者：有历史依据吗？
2. 原之者：有实际证据吗？
3. 用之者：有实践效果吗？
```

**可能的连接**：验证的三个维度。

**待验证**：这和现有的验证方法有什么不同？

---

## 法家：规则与理解的张力

规则可以强制行为，但无法生成理解。
完全依赖规则危险，完全依赖理解也危险。

**可能的连接**：Skill 是规则形式的理解，而非理解形式的规则。

**待验证**：这句话什么意思？我真的理解吗？

---

## 三层模型

```
道（原则）→ 生成 → 术（方法）→ 具体化 → 法（规则）

但传递只能反向：
法 → 暗示 → 术 → 指向 → 道
```

文档只能传"法"，接收者必须自己重建"术"和"道"。

**待验证**：这个模型有用吗？怎么应用？

---

## 如何消化

1. 在实际工作中遇到相关情况时，回来看看
2. 问自己：这个古人的话，在这个具体场景下是什么意思？
3. 如果找到真正有用的连接，再加入 wisdom-distilled.md

不要急于"用起来"。读到 ≠ 内化。
