# Orientation Examples

## Example 1: Node.js Project (Healthy)

```markdown
## Project Overview

CLI tool for managing cloud infrastructure, built with TypeScript.

## Key Entry Points

- **CLAUDE.md**: Testing requirements, code conventions
- **README**: Installation, usage examples

## Available Skills

| Skill    | When to use                    |
| -------- | ------------------------------ |
| dive     | Investigate codebase questions |
| refining | Commits and PRs                |

## Project Type

- Stack: TypeScript, Node.js
- Notable: GitHub Actions CI, Jest tests

## Suggested Starting Points

1. `src/index.ts` for CLI entry point
2. `dive` skill to investigate specific features
```

## Example 2: Monorepo (Docs Need Attention)

```markdown
## Project Overview

Monorepo with frontend (React), backend (Python), shared infra.

## Key Entry Points

- **README.md**: Basic overview only

## Documentation Health

⚠️ No CLAUDE.md or AGENTS.md found. Future agents will lack guidance.
**Recommendation**: Use `housekeeping` to create agent docs.

## Project Type

- Stack: React, Python, PostgreSQL
- Structure: Monorepo (pnpm workspaces)
- Notable: Docker Compose, Turborepo

## Suggested Starting Points

1. Root `package.json` for workspace config
2. Consider creating AGENTS.md
```
